# agent.py 教学视频脚本

## 00:00 - 00:10 开场

今天我们不逐行硬啃 `agent.py`，而是把它看成一个 agent 的总调度中心。你只要抓住一条主线：用户输入进来，模型思考，工具执行，结果回到模型，最后给用户答案。

## 00:10 - 00:22 文件地图

这个文件有四块：第 1 到 222 行是导入和兼容入口，第 223 到 238 行是测试模型和模型构造入口，第 241 到 4160 行是 `LearningAgent` 主类，第 4163 到 4229 行是权限和 CLI 入口。真正要细读的是 `LearningAgent` 内部的小功能段。

## 00:22 - 00:36 初始化

第 242 到 352 行是初始化和保存状态。这里保存模型、工作区、权限函数、工具注册表、MCP 状态、桌面控制器、任务记录、记忆文件和日志路径。它像开工前把工具箱摆好。

## 00:36 - 00:52 主循环

第 454 到 636 行是 `run_events()`，也就是核心主循环。它创建 session，构造 messages，请求模型，接收模型回答。如果模型要用工具，就执行工具，把结果再喂回模型。这个循环一直跑到模型给出最终回答。

## 00:52 - 01:06 工具调用

第 879 到 1009 行决定模型能看到哪些工具。第 1383 到 1542 行负责真正执行工具和 MCP 调用。模型不能直接读写电脑，它只能发出工具请求，agent 负责判断是否可见、是否有风险、是否需要权限。

## 01:06 - 01:20 MCP

第 1513 到 1542 行是 MCP 工具调用主入口，第 1566 到 1723 行是 MCP stream、resource 和 prompt 读取。MCP 像给 agent 插上外部能力的插槽，但仍然由 agent 做审批、调用和记录。

## 01:20 - 01:34 权限和安全

第 4163 到 4229 行是终端权限和启动入口。写文件、跑命令、控制桌面都不是自动放行的。权限函数会把请求变成结构化事件，让用户和验收系统都知道：谁想做什么，参数是什么，最后是否允许。

## 01:34 - 01:48 长任务能力

第 1769 到 2202 行是任务和团队协作，第 2669 到 2756 行是计划模式，第 3955 到 4145 行是后台命令。这些段落让 agent 能处理更长、更复杂的目标。

## 01:48 - 02:00 总结

所以读 `agent.py` 的秘诀不是背每一行，而是先按行号地图跳读：242 到 352 行看状态，454 到 636 行看主循环，879 到 1542 行看工具，1769 到 2202 行看长任务，3399 到 3767 行看桌面控制。
