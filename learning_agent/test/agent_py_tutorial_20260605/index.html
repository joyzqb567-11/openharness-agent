00:00 / 代码地图课

把 agent.py 看成一个总调度中心

用户输入进来，模型思考，工具执行，结果回到模型，最后给用户答案。

这节课不逐行硬背，而是先建立能读懂源码的脑内地图。
00:10 / 文件结构

3692 行代码，先分成四块看

导入区接入 core、models、mcp、tools、browser、app 等模块。
测试模型FakeModel 给单元测试用，不联网，按预设返回。
LearningAgent主类，从第 239 行到 3626 行，是真正的调度中心。
权限和入口终端授权、客户模式、HTTP bridge、main 启动入口。
00:24 / 初始化

__init__ 不是回答问题，而是在摆工具箱

self.model = model self.workspace = Path(workspace).resolve() self.ask_permission = ask_permission self.mcp_tool_registry = McpToolRegistry() self.computer_use_controller = ComputerUseController()

它保存模型、工作区、权限函数、MCP 注册表、桌面控制器、任务记录、记忆文件和日志路径。

00:38 / run_events 主循环

一条用户请求，在这里被跑完整

用户输入创建 session 和 run id。
构造 messages装配系统提示词、记忆和用户问题。
请求模型模型决定回答，或请求工具。
执行工具读文件、跑命令、MCP、桌面控制。
最终回答保存摘要并发出完成事件。
核心记忆：run_events = 用户请求、模型响应、工具结果、最终回答的事件流发动机。
00:56 / Tool Loop

模型负责“想”，工具负责“做”

模型返回 ToolCall: name = "read_file" arguments = {"path": "learning_agent/core/agent.py"} agent 执行工具后，把结果放回 messages。

工具不是全部一次性暴露。很多工具要通过 tool_search 或 skill_read 才解锁，这样模型不容易被太多工具带跑。

01:10 / MCP 外部工具

MCP 像给 agent 插外部能力的插槽

启动 server启动前先走权限确认。
收集 schema把外部工具转成模型能看懂的格式。
调用工具执行前判断风险和参数。
记录进度成功、失败、401、stream 都进入观察记录。
结果回灌工具结果再给模型继续推理。
01:24 / 权限和安全

模型可以建议操作，但不能默默改电脑

写文件write、edit、notebook_edit 都需要边界。
跑命令bash 和后台命令必须先问。
MCP 工具外部世界工具按风险审批。
桌面控制Computer Use 动作要观察、审批、记录证据。
01:38 / 长任务能力

它不只会一问一答，还能记录、拆分和恢复

task / task_list / task_get team_create / send_message enter_plan_mode / verify_plan_execution start_background_command / read_background_command

这些功能让 agent 能处理更长目标：先计划、再执行、能看进度、能停下来、还能从会话摘要恢复。

01:50 / 读源码顺序

以后读它，按这个顺序

1. __init__：准备哪些能力 2. run_events：一次请求怎么跑完 3. _build_initial_messages：模型看到什么 4. _available_tool_schemas / _execute_tool：工具怎么出现和执行
一句话总结：agent.py 不是模型本身，它是让模型安全、可观察、可恢复地使用工具的运行时外壳。
