**总体设计意图（高层）**
- 意图：实现一个教学用途的最小私人 agent，展示模型适配层、工具循环、文件工具、OAuth/Codex 桥接、长期记忆和操作权限确认等基本组件，便于初学者理解完整的 agent 链路。
- 如果缺失：项目将缺少整体示例，学习者无法看到如何把模型、工具、安全边界和日志整合成一个可交互系统。

## Imports（依赖与用途）
- `from __future__ import annotations`
  - 意图：延迟类型注解解析，允许在类型注解中引用尚未定义的类或使用更简单的字符串化类型，提升向后兼容性。
  - 如果缺失：当存在跨引用类型注解或 Python 版本较旧时，可能导致 NameError 或需要用字符串形式显式写类型；开发体验下降，但运行时通常无影响（取决于 Python 版本）。

- `import base64, hashlib, http.server, json, os, secrets, subprocess, tempfile, time, urllib.*, webbrowser, dataclasses, pathlib.Path, typing...`
  - 意图：分别提供 PKCE/JWT 编解码、OAuth 本地回调、JSON 处理、环境配置读取、安全随机、调用外部 CLI、临时目录、时间、HTTP 请求、打开浏览器、轻量数据对象（dataclass）、路径处理和类型约束等低层能力，构成 agent 的运行时工具箱。
  - 如果缺失：agent 的某些功能会直接失效，例如没有 `http.server` 无法接收 OAuth 回调；没有 `subprocess` 无法调用 Codex CLI；没有 `json`/`urllib` 无法与 HTTP API 交互。每个 import 对应一个可观测功能缺失。

## 数据类和轻量类型（`ToolCall`, `ModelMessage`, `ChatModel`）
- `ToolCall`
  - 意图：作为模型请求“调用工具”的标准内部表示，携带工具名、参数和唯一 call_id，用于把模型层与工具执行层解耦并追踪每次调用。
  - 如果缺失：无法可靠将模型输出的“工具调用”转译成可执行的操作，也无法为每次工具调用生成可追踪 ID，调试和日志记录变得困难，系统会退化成只能直接返回文本的模型代理。

- `ModelMessage`
  - 意图：统一模型返回的结构，包含 `decision_note`（给人看的决策说明）、`text`（最终文本）和 `tool_calls`（待执行工具），便于不同模型适配器之间共享协议。
  - 如果缺失：模型适配器之间的接口会分裂，后续代码需要对不同返回格式做分支处理，增加维护成本和出错概率。

- `ChatModel` (Protocol)
  - 意图：定义模型适配器必须实现的 `chat` 方法接口，使得可以注入假模型、OpenAI、Codex CLI、Codex OAuth 等多种实现并互换。
  - 如果缺失：丧失静态接口约束，增加误用风险（不同适配器可能实现不一致的签名），测试注入不方便。

## TOOL_SCHEMAS（向模型声明工具能力的 JSON schema）
- 意图：向模型声明 agent 可用工具的名字、说明和参数 schema。严格的 schema 能让结构化模型（或使用 `--output-schema` 的后端）以稳定格式返回工具调用，减少解析错误。
- 如果缺失：模型不知道该如何请求工具或以什么格式请求；即使模型尝试自由格式输出，解析层也更容易出错；会降低自动化交互的可靠性。

## 假模型 `ToolCallingFakeModel`
- 意图：在单元测试或教学演示中替换真实模型，按预设返回序列控制 agent 行为，避免网络依赖或真实模型费用。
- 如果缺失：测试和教学演示将更依赖真实网络模型，难以做可复现的单元测试。

## 真实模型适配器（`OpenAIChatModel`, `CodexCliChatModel`, `CodexOAuthChatModel`）
- `OpenAIChatModel`
  - 意图：包装常见的 OpenAI API 客户端调用，并把 SDK 返回标准化为 `ModelMessage`。延迟导入 `openai` 让测试和假模型不强依赖该包。
  - 如果缺失：无法直接使用 OpenAI API；若直接在主流程调用 SDK，测试隔离性会更差。

- `CodexCliChatModel`
  - 意图：支持通过本地 `codex exec` 可执行文件作为离线/本地桥接，以 `--output-schema` 和文件形式实现结构化输出，并在不联网或习惯用 Codex CLI 的场景下替代远程 API。
  - 如果缺失：没有本地桥接路径给到使用者，面对 network-restricted 环境或偏好本地工具的用户时缺少选择；同时一些教学场景（演示 CLI 流程）无法复现。

- `CodexOAuthChatModel`
  - 意图：提供使用 OpenAI/Codex OAuth + Responses API 的直连实现，含 PKCE、浏览器登录、本地回调、token 存储/刷新和 SSE/流式响应解析等完整链路，方便在需要直接调用 Codex Responses 的场景使用。
  - 如果缺失：无法示范 OAuth 完整登录与 token 管理；对需要直连 Codex Responses 的高级用户无可行方案。

每个适配器中还有很多容错、延迟导入、错误信息包装的细节，意图在于把网络、子进程和解析错误转换成用户可读的 ModelMessage，而不是抛出原生异常终止程序。

---

## Codex CLI / OAuth 内部细节（构造命令、schema、SSE 解析、JWT 提取）
- 意图：这些函数分别负责把 agent 的 messages/tools 包装为平台要求的提示词或请求体、构造子进程参数、读取输出文件、解析流式 SSE 事件、以及从 id_token/access_token 提取 account_id。技术上它们把平台差异封装起来，提供统一的 ModelMessage 输出。
- 如果缺失：跨平台适配（CLI 与 OAuth）会变得脆弱，或需要在主逻辑中硬编码各后端的细节，增加耦合和错误面；例如没有 SSE 解析将无法从 Responses API 的流式数据中重建最终文本。

---

## Token 存储 (`CodexOAuthTokenStore`) 与 Token 对象
- 意图：把 OAuth token 持久化到用户本地可配置路径，避免把凭据写入仓库；同时提供加载/保存的健壮逻辑以应对损坏文件。
- 如果缺失：token 只能保存在内存中，会话断开后需要频繁登录；或不小心将 token 写入代码仓库，造成安全风险。

## Agent 核心：`LearningAgent`（交互循环与工具执行）
这是文件中最关键的部分，直接决定 agent 的行为边界。

- `__init__`（保存参数、初始化路径、确保 memory 文件存在）
  - 意图：保存模型适配器、工作区、权限回调和调试日志相关配置；确保工作区和 memory 文件存在以便后续读取和追加。
  - 如果缺失：agent 可能在运行中找不到 memory.md 或 debug_logs 路径；没有 ask_permission 回调会导致无法做有副作用的写入确认，存在安全隐患。

- `run`（主循环）
  - 意图：实现模型-工具的交互循环：向模型发送 messages+工具定义，让模型返回文本或工具调用，执行工具并把结果送回模型，直到模型不再请求工具或达到循环上限。加入调试事件记录让学习者追踪每一轮的输入/输出/工具调用。
  - 如果缺失：没有这个循环，agent 就不能进行多轮工具交互；如果不记录日志，学习者无法回放模型决策和工具执行链路，教学价值大幅下降。

- 工具执行与安全边界 (`_execute_tool`, `_read_file`, `_write_file`, `_append_memory`, `_resolve_workspace_path`)
  - 意图：把工具操作限制在工作区内、在写入前询问用户权限、对异常和越界路径做防护，并对长文件内容做截断返回，保证安全和可控性。
  - 如果缺失：模型可能任意读写磁盘、覆盖敏感文件或越界访问；没有权限确认会导致自动化写操作带来破坏性后果；没有路径解析和越界检查会有严重安全风险。

- 权限确认函数 `ask_permission_from_terminal`
  - 意图：提供一个简单的命令行交互实现，教学场景下让用户决定有副作用的操作是否执行。
  - 如果缺失：agent 无法得到用户确认，任何 write/append 请求要么自动执行（危险），要么被阻塞。

## 日志/调试（`_write_debug_event` 与 Markdown 可读日志）
- 意图：把运行时事件结构化地写入 JSONL，并同时维护一个可读的 Markdown 日志，方便学习者与开发者回溯每一步决策、工具调用与结果。
- 如果缺失：调试变得困难，无法方便地把某次失败或不正确行为回放定位；教学效果和可解释性显著下降。

---

## 辅助函数（prompt 构造、schema 刷新、SSE 解析等）
- 意图：把繁琐细节（例如 `--output-schema` 要求、prompt 的具体措辞、结果解析策略）封装在一个函数里，保持主流程清晰，并让不同模型适配器可以复用同样的解析逻辑。
- 如果缺失：主流程会被大量平台特定逻辑污染，代码可读性与可维护性降低。

## `main()` 入口
- 意图：在文件直接运行时提供一个交互式命令行 demo，使用环境变量切换模型实现，适合本地试验与教学演示。
- 如果缺失：没有交互 demo，学习者需要自己编写运行时脚本才能体验 agent 行为，门槛提高。
